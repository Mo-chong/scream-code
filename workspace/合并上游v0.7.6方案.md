## 二、重叠文件分析（关键风险区）

本地二开改动与上游重叠的 **9 个文件**：

| 文件 | 上游改动 | 本地改动 | 风险等级 |
|------|---------|---------|---------|
| `package.json` | 版本号 0.7.5→0.7.6 | 版本号保持 0.7.2 | **低** — 仅版本号差异 |
| `apps/scream-code/package.json` | 版本号 0.7.5→0.7.6 | 版本号保持 0.7.2 | **低** — 仅版本号差异 |
| `apps/scream-code/src/cli/run-prompt.ts` | import 路径修正 | TUI 继承性改动（import 少） | **中** — 需 merge 确认 |
| `apps/scream-code/src/cli/run-stream-json.ts` | import 路径修正 | 本地有改动 | **中** — 需 merge 确认 |
| `apps/scream-code/src/tui/scream-tui.ts` | /model diy bindings 入口 | 大规模本地 TUI 改动（~50 文件） | **高** — 最大风险 |
| `apps/scream-code/test/cli/run-prompt.test.ts` | 新增测试 | 本地也有改动 | **中** |
| `apps/scream-code/test/tui/scream-tui-startup.test.ts` | 新增测试 | 本地也有改动 | **中** |
| `packages/agent-core/src/session/subagent-host.ts` | /model diy bindings | 本地有改动 | **高** — 核心逻辑 |
| `packages/agent-core/test/session/subagent-host.test.ts` | 新测试 | 本地有改动 | **中** |

### 🚨 高优先级冲突

#### 1. `subagent-host.ts` ⚠️ 上游修复 /model diy bindings 调用链
上游改动：将 `setSubagentModelBindings` hook 正确路由到子 agent spawn 路径。

本地改动：此文件包含二开的功能扩展（记忆系统、Bash 降噪等）。待确认 diff 内容决定是：
- 接受上游 + 重加本地改动
- 或将上游小修 backport 到本地

**操作方法**：merge 时以本地为准，手动 cherry-pick 上游的 bindings 逻辑。

#### 2. `scream-tui.ts` ⚠️ 上游新增 /model 命令的 TUI 集成
上游改动：增加 subagent model binding 相关的 TUI hook。

本地改动：此文件大量重写（commands/controllers/managers 等 50+ 文件）。merge 冲突极多。

**操作方法**：merge 时以本地为准，手动将上游 /model diy 相关的 TUI 集成点提取出来。

## 三、合并方案（推荐：Origin Push → Merge → Test）

### 方案流程图

```
step 1: 提交或 stash 本地未提交改动
step 2: git pull origin main                 # 拉取上游 v0.7.2→v0.7.6 全量变更
step 3: 解决 merge conflict（重点是 9 个重叠文件）
step 4: guard 脚本验证（alwaysBundle / bundle-stale）
step 5: pnpm install （lock 可能变）
step 6: pnpm build （两段构建链）
step 7: 确认 build 通过 + 关键功能正常
step 8: git push mochong main                 # 推回二开仓库
```

### 具体操作

#### Step 1 — 保存当前状态

```bash
cd /d/D:/AI/ScreamCode

# 如果有未提交改动
git stash push -m "pre-merge-v0.7.6-$(date +%Y%m%d)"
# 或提交：
git add -A && git commit -m "chore: save state before merge v0.7.6"
```

#### Step 2 — 拉取上游

```bash
git pull origin main
# Git 会自动尝试三方合并
# 基点 v0.7.2 → 合并 v0.7.2..v0.7.6 到当前 HEAD
```

#### Step 3 — 解决冲突（手动）

预计以下文件会出现冲突：

**高优先级（手动编辑）：**

1. **`packages/agent-core/src/session/subagent-host.ts`**
   - 策略：以上游为准解冲突，重加本地改动
   - 操作：`git checkout --ours <冲突段>` 保留上游主体 → 再手动加回本地记忆系统/Bash降噪改动

2. **`apps/scream-code/src/tui/scream-tui.ts`**
   - 策略：以本地为准，手动提取上游 /model diy 集成
   - 操作：`git checkout --ours` 保留本地 → 再读取上游 diff 手动补入 /model bindings 的 TUI 集成点

**低优先级（无争议）：**

3. `package.json` → 版本号改 `0.7.6`（接受上游）
4. `apps/scream-code/package.json` → 版本号改 `0.7.6`（接受上游）
5. `run-prompt.ts` / `run-stream-json.ts` → 按冲突情况逐一处理
6. 测试文件 → 按冲突逐一处理

#### Step 4 — Guard 验证

```bash
# 必须先验证 alwaysBundle 没被覆盖
bash scripts/guards/guard-always-bundle.sh

# 验证构建链配置完整
bash scripts/guards/guard-config-build.sh

# 验证全部 4 个 guard
bash scripts/guards/check-all.sh
```

#### Step 5 — 安装依赖

```bash
pnpm install
```
> ⚠️ `pnpm-lock.yaml` 上游 v0.7.2→v0.7.6 间可能有变更，pnpm install 会自动合并

#### Step 6 — 构建

```bash
bash scripts/build-dev.sh
```
> ⚠️ 记忆踩坑：build-dev.sh 硬编码 node 路径直调 tsdown，不走 pnpm lifecycle，适合 Windows Git Bash

#### Step 7 — 功能确认

```bash
# 编译通过确认
echo "Build OK"

# 核心功能快速检查
node -e "const m=require('./packages/agent-core');console.log('agent-core loaded')"

# guard-bundle-stale 确认源码与产物匹配
bash scripts/guards/guard-bundle-stale.sh
```

#### Step 8 — 推回二开仓库

```bash
git push mochong main
```

## 四、风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| `scream-tui.ts` 合并冲突复杂 | 高 | 高 | 以本地为准，手动 cherry-pick 上游 /model diy 集成 |
| `subagent-host.ts` 冲突破坏二开功能 | 中 | 高 | merge 后 run `bash scripts/guards/check-all.sh` |
| `pnpm-lock.yaml` 合并后不一致 | 中 | 中 | `pnpm install` 自动修复 |
| guard-bundle-stale 检测到未编译 | 低 | 中 | 运行 build-dev.sh 即可 |
| 上游改了什么导致 tsc 编译失败 | 低 | 高 | 按 error 修复；如有 v0.7.6 新增接口需要适配回本地 |
| merge 后 alwaysBundle 被覆盖 | 低 | 高 | guard-always-bundle.sh 会拦截 git commit |

## 五、踩坑回顾（来自记忆）

| 踩坑 | 方案 |
|------|------|
| pnpm build --filter 触发 prepare 脚本需 node → Git Bash PATH 无 node | ✅ `scripts/build-dev.sh` 硬编码 node 路径直调 tsdown |
| tsdown shell wrapper on Windows 不能直接通过 node 调用 | ✅ 直调 `tsdown/dist/run.mjs` |
| 只 build agent-core 不 build scream-code → alwaysBundle 覆盖不到 | ✅ `build-dev.sh` 两段构建链：agent-core → scream-code |
| merge 后 alwaysBundle 配置可能被覆盖 | ✅ `guard-always-bundle.sh` guard 脚本拦截 |
| 上游改了 @scream- 包名 → 本地 import 路径失效 | 需手动 check 所有 @scream- import 在 merge 后是否正确 |
